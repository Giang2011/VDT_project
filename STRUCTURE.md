# 📁 Project Structure

> **IMPORTANT — Agent note:** All source files MUST be created at the exact paths defined in this document.
> Do not create files outside the defined tree. Do not rename directories.
> This file is the single source of truth for project layout.

---

## Full Directory Tree

```
monitoring-agent/                          # ← project root
│
├── src/                                   # All Python source lives here
│   └── monitoring_agent/                  # Installable Python package
│       ├── __init__.py                    # Package init (version string)
│       ├── main.py                        # Entry point — starts loop + watcher
│       ├── config.py                      # Loads and validates config.yaml
│       ├── logger.py                      # Configures file logger + syslog handler
│       ├── alerter.py                     # Checks metrics against thresholds; emits warnings
│       └── collectors/                    # Sub-package: one module per resource type
│           ├── __init__.py                # Exposes collect_all() convenience function
│           ├── cpu.py                     # CPU % usage, load average
│           ├── memory.py                  # RAM total/used/percent
│           ├── disk.py                    # Disk usage per partition + I/O counters
│           ├── network.py                 # bytes_sent, bytes_recv, packets per NIC
│           └── fs_watcher.py              # watchdog Observer + event handler
│
├── config/                                # Config templates (not secrets)
│   ├── config.yaml.example               # Fully commented reference config
│   └── logrotate.conf                     # logrotate rule for agent log file
│
├── systemd/                               # systemd integration
│   └── monitoring-agent.service           # Unit file (Type=simple, Restart=on-failure)
│
├── packaging/                             # Build & packaging scripts
│   ├── build_deb.sh                       # Builds .deb via fpm
│   ├── build_rpm.sh                       # Builds .rpm via fpm
│   ├── postinstall.sh                     # Runs after package install (enable service)
│   └── preremove.sh                       # Runs before package remove (stop service)
│
├── tests/                                 # pytest test suite
│   ├── __init__.py
│   ├── conftest.py                        # Shared fixtures (mock psutil, tmp config)
│   ├── test_config.py                     # Config loading / validation
│   ├── test_collectors.py                 # CPU, memory, disk, network collectors
│   ├── test_alerter.py                    # Threshold logic
│   ├── test_logger.py                     # Logger setup, file + syslog handlers
│   └── test_fs_watcher.py                 # FS event handler (using tmp directories)
│
├── docs/                                  # Design & deployment documentation
│   ├── design.md                          # System design, architecture decisions
│   └── deploy.md                          # Build, install, and demo guide
│
├── dist/                                  # Output directory for built packages (git-ignored)
│   ├── monitoring-agent_1.0.0_amd64.deb  # (generated by build_deb.sh)
│   └── monitoring-agent-1.0.0.x86_64.rpm # (generated by build_rpm.sh)
│
├── .venv/                                 # Python virtual environment (git-ignored)
├── setup.py                               # setuptools packaging config
├── pyproject.toml                         # Build system config (PEP 517)
├── requirements.txt                       # Runtime dependencies (pinned)
├── requirements-dev.txt                   # Dev/test dependencies
├── .flake8                                # Linter config
├── .gitignore
├── README.md                              # Project overview and quick-start
├── PLAN.md                                # Detailed technical plan
├── STRUCTURE.md                           # This file — canonical directory layout
├── TASKS.md                               # Agent task list derived from PLAN.md
└── LICENSE
```

## Key Path Conventions

| Concern | Path |
|---|---|
| Python source | `src/monitoring_agent/` |
| Collector modules | `src/monitoring_agent/collectors/` |
| Config template | `config/config.yaml.example` |
| systemd unit | `systemd/monitoring-agent.service` |
| Build scripts | `packaging/` |
| Tests | `tests/` |
| Built packages | `dist/` ← git-ignored |
| Runtime config (installed) | `/etc/monitoring-agent/config.yaml` |
| Runtime logs (installed) | `/var/log/monitoring-agent/agent.log` |

---

## Naming Conventions

| Item | Convention | Example |
|---|---|---|
| Python files | `snake_case.py` | `fs_watcher.py` |
| Python classes | `PascalCase` | `AgentFSHandler` |
| Python functions | `snake_case` | `collect_all()` |
| Config keys | `snake_case` | `interval_seconds` |
| Shell scripts | `snake_case.sh` | `build_deb.sh` |
| Package name | `kebab-case` | `monitoring-agent` |
| systemd service | `kebab-case.service` | `monitoring-agent.service` |

---

## Files That Must Not Be Created

- Do **not** put Python source directly under project root (use `src/monitoring_agent/`)
- Do **not** create a `lib/` or `app/` directory — use `src/` only
- Do **not** create per-collector config files — single `config.yaml` only
- Do **not** create a `scripts/` directory — packaging scripts live in `packaging/`
